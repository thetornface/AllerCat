# Подключение формы предзаказа к Google Sheets

Сайт статический (Astro → GitHub Pages), бэкенда у него нет.
Самый простой и бесплатный способ записывать заявки в чужую
Google-таблицу — Google Apps Script Web App. Один раз настраиваете, дальше
работает без вмешательства.

## Что нужно сделать вам один раз (≈ 5 минут)

1. Откройте таблицу:
   <https://docs.google.com/spreadsheets/d/1rS4YIFD29kbu3SKJBgbn9ZdxhOHneZ_2_mqFujIXtAc/edit>

2. В меню выберите **Расширения → Apps Script**. Откроется редактор.

3. Удалите содержимое файла `Code.gs` и вставьте код из
   [`preorder-handler.gs`](./preorder-handler.gs). Сохраните (Cmd+S).

4. В редакторе нажмите **Развернуть → Новое развертывание**.

   - Тип: **Веб-приложение**
   - Описание: например, `AllerCat preorder v1`
   - Запускать от имени: **Я (ваш аккаунт)**
   - У кого есть доступ: **Все** (Anyone) — это нужно, чтобы форма с
     сайта могла достучаться до скрипта без авторизации.

   Нажмите **Развернуть**, разрешите запрашиваемые права (доступ к
   таблице).

5. Скопируйте полученный URL вида
   `https://script.google.com/macros/s/AKfyc.../exec`.

6. Откройте в проекте файл `src/pages/index.astro`, найдите строку:

   ```js
   const PREORDER_ENDPOINT = 'https://script.google.com/macros/s/REPLACE_WITH_YOUR_DEPLOYMENT_ID/exec';
   ```

   и подставьте свой URL вместо плейсхолдера. Сохраните, закоммитьте и
   задеплойте сайт. Готово.

## Как проверить

- Откройте URL развертывания напрямую в браузере — должен прийти JSON
  `{"ok":true,"service":"AllerCat preorder handler"}`. Если прилетел
  HTML с ошибкой — значит, права доступа не разрешены (см. шаг 4).
- На сайте откройте форму предзаказа, заполните и отправьте. В таблице
  должна появиться новая строка с датой, именем, контактом, соцсетями,
  адресом страницы и user-agent браузера.

## Если потом надо обновить логику

Меняете код в `preorder-handler.gs`, вставляете в редактор, выбираете
**Развернуть → Управление развертываниями → ✏️ → Новая версия →
Развернуть**. URL сохранится, ничего на сайте трогать не придётся.

## Безопасность и приватность

- Apps Script бежит от вашего Google-аккаунта; владеть таблицей и
  скриптом будете только вы.
- Скрипт принимает только нужные поля и обрезает их по длине, чтобы
  никто не залил в таблицу мегабайтный мусор.
- Запросы с сайта идут с `Content-Type: text/plain;charset=utf-8` —
  это специально, чтобы браузер не делал CORS-preflight, который Apps
  Script плохо обрабатывает.
- Запрос отправляется в режиме `no-cors`, поэтому фронтенд не читает
  ответ скрипта. Это нормально: пользователю всё равно показываем
  «Спасибо», а ошибки серверной стороны — в логах Apps Script
  (Расширения → Apps Script → Журналы выполнения).

## Защита от спама (опционально, если нужно)

Если будут спамить, можно:

1. Добавить в `doPost` проверку на минимальную длину поля `name` и
   формат `contact` (regex для email/телефона).
2. Поставить простой honeypot на форму (скрытое поле, которое люди
   не заполняют, а боты — да).
3. Подключить Cloudflare Turnstile / Google reCAPTCHA v3 — бесплатно,
   но требует пары лишних шагов (надо ещё один endpoint для проверки).

Пока заявок будет немного — не трогайте, и так норм.
